# YouTube Tracker — Domain Glossary

## Core Terms

**Personal Knowledge Base**
The primary purpose of this app. A local tool for a single user to search and chat across YouTube channels they follow for research. Not an audit tool, not a multi-user product.

**Channel**
A YouTube channel being tracked. Identified by `youtube_channel_id`. The app polls each tracked channel on a schedule to discover new videos.

**Video**
A YouTube video ingested into the system. Identified by `youtube_video_id` (the canonical key — used for deduplication via UNIQUE constraint). Stores metadata, transcript status, and summary status.

**Transcript**
The full text of a video's spoken content, stored as a `.txt` file on disk. Source is either `youtube-transcript` (community library) or manual paste. Timestamps are preserved per segment when available.

**TranscriptProvider**
The interface that abstracts transcript acquisition. Primary implementation: `youtube-transcript` library. Fallback: manual paste/upload. Other implementations (yt-dlp, Whisper) can be added without touching the pipeline.

**Transcript Segment**
One raw unit returned by the transcript provider — typically 1–5 seconds, 5–15 words, with a start timestamp. Segments are accumulated into Chunks.

**Chunk**
A unit of transcript text sized for embedding and retrieval (~500 tokens). Built by accumulating Transcript Segments up to the token limit, cutting at the nearest sentence boundary, and carrying the start timestamp of the first segment. Overlaps the last 1–2 sentences with the next chunk.

**Embedding**
A vector representation of a Chunk's text, generated by Chroma's default model (`all-MiniLM-L6-v2`) internally on the Chroma server. The Node backend sends raw text; Chroma handles embedding.

**Ingestion Job**
A background task that processes one video or one channel sync through the full pipeline: metadata → transcript → chunks → Chroma index → summary. Jobs are persisted in SQLite and re-hydrated on server restart.

**Ingestion Pipeline**
The ordered sequence: fetch metadata → fetch transcript → save `.txt` → chunk → store chunks in SQLite → index in Chroma → generate short summary + key topics.

**Summary (eager)**
Generated at ingestion time: short summary and key topics. Ready when the video is first viewed.

**Summary (lazy)**
Generated on demand when the Video Detail page is opened: detailed summary, action items, technical terms, notable quotes.

**Channel Sync**
The process of fetching a channel's recent uploads and ingesting any video whose `youtube_video_id` does not yet exist in the `videos` table. Runs automatically on a 24-hour schedule (configurable). Deduplication is ID-based, not timestamp-based.

**Transcript Status**
One of: `pending` | `available` | `unavailable` | `failed`.
- `unavailable` — set immediately when YouTube API reports `hasCaptions = false`. No extraction attempted.
- `failed` — set when extraction threw unexpectedly. Retried once on the next sync.
- `available` — transcript successfully retrieved and stored.
- `pending` — not yet attempted.

**Search Scope**
The set of channels a Chat or Search query is restricted to. Defaults to the user's last-used selection, persisted in frontend state. Not stored in the database.

**Reindex**
A CLI command (`npm run reindex`) that rebuilds the Chroma collection from SQLite chunks + transcript files. Used to recover from Chroma/SQLite drift during local development.
